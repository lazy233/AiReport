## 数据库变更记录（与本仓库学生主档同步）

**日期**：2026-04-17。

上文「通用字段」中此前未落库的三项已补齐，并与 API `profile.basic` 键对应如下。

| 需求字段（文档名） | PostgreSQL 列名 | 表 | API `profile.basic` 键 | 说明 |
| ------------------ | --------------- | -- | ---------------------- | ---- |
| `advisor`（教服老师） | `advisor_teacher` | `students` | `advisorTeacher` | 与 `planner_teacher` 同属学生主档 |
| `grade_intake`（年级/入学时间） | `grade_intake` | `student_term_profiles` | `gradeIntake` | 与 `grade_level`（年级）并存；可填如 `2025级`、`2025/09` |
| `product`（产品） | `service_product` | `student_term_profiles` | `product` | 服务产品名；列名避免与 SQL 保留字混淆 |

**迁移方式**：应用 `init_db` 在 `Base.metadata.create_all` 之后，对已有数据库执行：

- `ALTER TABLE students ADD COLUMN IF NOT EXISTS advisor_teacher VARCHAR(64) NOT NULL DEFAULT ''`
- `ALTER TABLE student_term_profiles ADD COLUMN IF NOT EXISTS grade_intake VARCHAR(128) NOT NULL DEFAULT ''`
- `ALTER TABLE student_term_profiles ADD COLUMN IF NOT EXISTS service_product VARCHAR(256) NOT NULL DEFAULT ''`

新库由 SQLAlchemy 模型直接建表即包含上述列。若运行环境 PostgreSQL 过旧不支持 `ADD COLUMN IF NOT EXISTS`，需手工执行同等 `ALTER`（列已存在则跳过）。
